# Archive — Sector-Specific Variant Files

These files are **not part of the active 4P3X Refractable Base OS™ build**.

They are preserved here as reference material from earlier variant development — primarily a fleet/dispatch/driver management system that was used as a source during the base OS refactor.

None of these files are imported by the active codebase and they do not affect the build.

## Contents

| Folder | Description |
|---|---|
| `sector_specific_pages/` | Pages from the fleet/dispatch/driver/compliance variant (not routed) |
| `sector_specific_modules/` | UI modules for fleet, safety, execution, navigation (not used) |
| `sector_specific_services/` | Service layer files for fleet, dispatch, drivers, federation, maps, routing |
| `sector_specific_engine/` | Dispatch orchestrator, routing engine, intel engines (not used) |
| `sector_specific_sql/` | SQL schemas from the fleet/safety variant and original Supabase setup |

## Why Keep Them?

These files demonstrate the **refractable** nature of this base OS — the same architecture was previously used for a completely different sector and can be again.

They serve as a reference for future variant development.
